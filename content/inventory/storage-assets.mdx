## Storage IDs from `/etc/pve/storage.cfg`

| Storage ID | Type | Path / source | Nodes | Content |
|------------|------|---------------|-------|---------|
| `local` | Directory | `/var/lib/vz` | `pve1`, `pve3` | `backup`, `iso`, `vztmpl` |
| `local-lvm` | LVM-thin | Local LVM | `pve1`, `pve3` | `images`, `rootdir` |
| `pve1-storage` | ZFS | Local ZFS on `pve1` | `pve1` | — |
| `pve2-storage` | (defined) | — | `pve2` | Inactive; cluster artefact |
| `nas` | NFS | `192.168.1.16:/data/pve` | `pve1`, `pve3` | `rootdir`, `iso`, `backup`, `images` |

NFS-specific options for `nas`:

- `prune-backups keep-all=1`

## ZFS pools

| Pool | Host | Role |
|------|------|------|
| `pve1-storage` | `pve1` | Application data on the infra node |

`pve3` runs no native ZFS pool. The TrueNAS VM owns the ZFS layer on the production node via passed-through disks.

## TrueNAS

- VM `127` `truenas` on `pve3`.
- Multiple physical disks passed through.
- Exposes datasets to the rest of the cluster (NFS/SMB are typical, exact share list not captured here).
