## Conventions

- VM IDs are allocated from per-role ranges. Ranges currently in use:
  - `100–199` — ingress and small platform VMs (on `pve1`).
  - `200–299` — application VMs (currently on `pve1`, target `pve3`).
  - `300–399` — media (on `pve3`).
  - `400–499` — edge / network (on `pve3`).
  - `10000+` — lab desktops.
- VM names match the service they host where there is a 1:1 mapping.
- Every VM should point to a service page under [Services](/services/core/reverse-proxy).

## Notes

VM `127` (`truenas`) is intentionally outside the per-role ranges because it predates the convention. Do not renumber.
