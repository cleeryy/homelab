## Placement

- VM `204` `ubuntu-n8n` on `pve1`.
- Beszel label `n8n`, IP `192.168.1.13`.
- Beszel reports the system hostname as `ntfy`, which does not match the Beszel label or the Proxmox VM name.

## Docker containers

| Container | Function |
|-----------|----------|
| `n8n-compose-n8n-1` | n8n application |
| `n8n-compose-traefik-1` | Internal reverse proxy in front of n8n |
| `portainer` | Container management UI |
| `watchtower` | Image auto-update |
| `beszel-agent` | Monitoring agent |

## Dependencies

- The stack includes its own Traefik. Whether traffic enters this Traefik directly or first transits the cluster-level reverse proxy on `pve1` is **not captured here**.
- n8n state (workflows, credentials) is stored locally in the n8n container's data directory.

## Operational notes

- `watchtower` periodically updates container images; recovery procedures should not assume image tags are pinned.
- The hostname mismatch (`ntfy` vs. `n8n`) is recorded so operators do not misidentify the host when reading Beszel.

## Failure impact

- A `pve1` outage takes the n8n host with it.
- Workflows that integrate with external systems will fail silently if downstream APIs are not reachable.
