## Components

| Kind | ID | Name | Host | Function |
|------|----|------|------|----------|
| LXC | `111` | `forgejo` | `pve1` | Git forge |
| LXC | `124` | `forgejo-runner` | `pve3` | CI runner |

## Placement

- The forge lives on `pve1` (infrastructure node).
- The runner lives on `pve3` (compute node). Keeping CI on `pve3` avoids loading the infra node with build work.

## Dependencies

- Forgejo uses a database. The PostgreSQL LXC (`112`) on `pve1` is the most likely backend, but the exact database wiring is not exported into this documentation.

## Storage

- Forge data and runner caches live on the LXC root disks (`local-lvm` on the respective hosts). Whether additional volumes from `pve1-storage` or `nas` are mounted is not captured here.

## Ingress

- Web UI is reachable via the reverse-proxy layer on `pve1`. The exact hostname / route is not captured here.

## Operational notes

- The runner depends on connectivity from `pve3` to the forge on `pve1`. A `pve1` outage halts CI even if `pve3` is up.
