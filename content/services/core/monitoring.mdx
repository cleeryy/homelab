## Components

| Kind | ID | Name | Host | Function |
|------|----|------|------|----------|
| VM | `202` | `ubuntu-beszel` | `pve1` | Beszel server (Docker host) |

The Beszel server VM runs the following Docker containers:

| Container | Purpose |
|-----------|---------|
| `beszel` | Beszel server |
| `beszel-agent` | Local agent on the Beszel host |
| `portainer` | Container management UI |
| `termix` | Application present on the host; specific role not documented |
| `watchtower` | Image auto-update for containers |

A `beszel-agent` container is also deployed on every Docker host VM. Proxmox hosts (`pve1`, `pve3`) report systemd-level metrics into Beszel as well.

## Observed at the last snapshot

- `pve1` and `pve3` both report no failed systemd services. `pve1` runs ~83 monitored services, `pve3` ~82.
- `pve1` shows higher memory pressure than `pve3`.
- `Jelly` reports high memory usage on the `jellyfin` container.
- `QBittorrent` reports `gluetun` as **unhealthy**.
- `Nextcloud` reports `portainer` as **restarting**.

These are transient observations; they should not be used to draw long-term conclusions.
