## Placement

- VM `301` `ubuntu-servarr` on `pve3`.
- Beszel label `Servarr`, IP `192.168.1.26`.

## Docker containers

| Container | Function |
|-----------|----------|
| `sonarr` | TV series automation |
| `radarr` | Movie automation |
| `prowlarr` | Indexer aggregator |
| `bazarr` | Subtitles |
| `jellysweep` | Jellyfin library maintenance utility |
| `flaresolverr` | Cloudflare challenge solver for indexers |
| `gluetun` | VPN client used by indexer/download traffic |
| `portainer` | Container management UI |
| `watchtower` | Image auto-update |
| `beszel-agent` | Monitoring agent |

## Dependencies

- The stack writes to a shared media library mounted from TrueNAS. The same library is consumed by Jellyfin (`300`) and produced by qBittorrent (`302`).
- `prowlarr` provides indexer configuration to `sonarr`, `radarr`, and `bazarr`.
- `gluetun` provides a VPN egress for traffic that should not exit on the host's public IP.

## Operational notes

- Network policy for the *arr containers depends on whether they are wired into `gluetun`'s network namespace. The exact wiring is **not captured here**.

## Failure impact

- A `pve3` outage takes the stack down alongside qBittorrent, Jellyfin, and TrueNAS.
- A `gluetun` failure may degrade indexer traffic but does not affect Jellyfin playback of already-downloaded content.
