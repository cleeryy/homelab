## Placement

- VM `300` `ubuntu-jelly` on `pve3`.
- Beszel label `Jelly`, IP `192.168.1.31`.

## Docker containers

| Container | Function |
|-----------|----------|
| `jellyfin` | Jellyfin media server |
| `seerr` | Request management (Jellyseerr / Overseerr family) |
| `beszel-agent` | Monitoring agent |

## Dependencies

- The media library is expected to live on TrueNAS datasets and to be mounted into the Jellyfin container. The exact mount path is **not captured here**.
- The media catalog is filled by the Servarr host (`301`), which writes to the same shared library.

## Operational notes

- `jellyfin` was reported as using a large amount of memory at the last observation. Transient.

## Failure impact

- A `pve3` outage takes Jellyfin, the Servarr stack, and qBittorrent down at the same time. TrueNAS, which sits on the same node, is also affected.
- Catalog continuity depends on Jellyfin's metadata volume, not on the media files themselves.
