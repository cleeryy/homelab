# Roadmap

## Court terme (Q3 2026)

- [ ] Finaliser la documentation existante (services, runbooks, backups)
- [ ] Automatiser les backups Docker avec scripts et alertes
- [ ] Déployer un service de monitoring (Uptime Kuma ou Netdata)
- [ ] Mettre en place un VLAN dédié pour les services exposés

## Moyen terme (Q4 2026)

- [ ] Mise en place d'un monitoring centralisé (Prometheus + Grafana)
- [ ] Automatisation des déploiements (CI/CD via Dokploy + Git)
- [ ] Migration des services critiques vers des stacks Docker Compose versionnées
- [ ] Documentation des procédures de recovery avancées

## Long terme (2027)

- [ ] Haute disponibilité des services critiques (cluster Proxmox)
- [ ] Infrastructure as Code (Terraform / Ansible pour le homelab)
- [ ] Lab de test isolé pour expérimentations

## Terminé

- [x] Mise en place du wiki de documentation (Fumadocs + Dokploy)
- [x] Infrastructure Proxmox de base (PVE-01 + TNAS-01)
- [x] Déploiement de Traefik comme reverse proxy
- [x] Stockage centralisé TrueNAS avec partages NFS
