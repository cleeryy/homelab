## Fiche service (template)

Pour documenter un nouveau service, utiliser le format suivant :

```mdx
---
title: "<Nom du service>"
description: "<Description brève>"
---

# <Nom du service>

## Objectif
À quoi sert ce service dans le homelab ?

## Accès
- **URL interne** : http://<host>:<port>
- **URL externe** : https://<domaine>
- **Authentification** : <méthode>

## Stack
- **Méthode de déploiement** : Docker / Bare-metal / VM
- **Image / version** : <image:tag>
- **Dépendances** : <services requis>

## Configuration
Points clés de configuration, volumes montés, variables d'environnement.

## Sauvegarde
Chemins et méthodes de sauvegarde des données du service.
```

## Conventions de déploiement

- **Docker** : tout service déployé en conteneur utilise Docker Compose ou Dokploy
- **Réseau** : Traefik fait office de reverse proxy unique, les services exposent via labels Traefik
- **Volumes** : les données persistantes sont montées sur un volume Docker ou un bind mount vers un datastore ZFS
- **Nommage** : noms de conteneurs en kebab-case, cohérents avec le nom du service
- **Tags** : privilégier les tags de version fixes (`:1.2.3`) plutôt que `:latest`
