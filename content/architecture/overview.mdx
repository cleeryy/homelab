## Cluster

- Proxmox VE cluster `home`, two active nodes, quorum enabled.
- `pve1` (`192.168.1.101`) is centered on infrastructure, management, ingress, and the database/code/wiki stack.
- `pve3` (`192.168.1.103`) is centered on production workloads: storage VM (TrueNAS), media stack, edge router, lab containers.
- `pve2` is referenced in cluster storage (`pve2-storage`) but is not active.

The cluster is **asymmetric** by usage: `pve3` carries the heavier compute and disk workload (large `local-lvm` plus a TrueNAS VM with attached disks), while `pve1` carries the higher number of small services and shows higher memory pressure.

## Layers

The homelab is organised in three execution layers:

1. **Proxmox host** — bare metal, two Debian/PVE 9.x nodes.
2. **VMs and LXC containers** — managed by Proxmox.
3. **Docker stacks inside VMs** — most user-facing apps run inside an Ubuntu 24.04 VM that hosts a `docker compose` stack alongside `portainer`, `watchtower`, and `beszel-agent`.

This three-layer model is reflected in the documentation: Proxmox-level placement lives under [Architecture](/architecture/service-placement) and [Inventory](/inventory/virtual-machines); per-stack detail lives on each service page.

## Building blocks

- **Edge / routing.** OPNsense VM (`401`) on `pve3`, with `net0` on `vmbr0` (management LAN) and `net1` on `vmbr1` (internal `10.0.0.0/24`).
- **Ingress.** Nginx Proxy Manager VM (`100`) on `pve1`. A second reverse-proxy VM `ubuntu-nginxrpm` (`105`) is also present on `pve1`; the exact split between the two is not fully confirmed.
- **Egress / tunnel.** `cloudflared` LXC (`103`) on `pve1`.
- **Code.** Forgejo LXC (`111`) on `pve1`, with the `forgejo-runner` LXC (`124`) on `pve3` for CI compute.
- **Database.** PostgreSQL LXC (`112`) on `pve1`. A second PostgreSQL LXC (`113`) is defined but stopped.
- **Storage.** TrueNAS VM (`127`) on `pve3` with multiple attached disks. Shared NFS `nas` from `192.168.1.16:/data/pve`. ZFS pool `pve1-storage` on `pve1`. `local-lvm` on each node for guest disks.
- **Observability.** Beszel VM (`202`) on `pve1` with `beszel-agent` deployed on every Docker host.
- **Application stacks.** Nextcloud, n8n, Jellyfin, the Servarr stack, qBittorrent, and a `web-apps` host — all are Ubuntu VMs running a Docker compose stack.
- **Lab / experimentation.** Several LXCs on `pve3` for code, EDR testing, and pre-production work.
