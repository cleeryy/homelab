## Proxmox storage from `/etc/pve/storage.cfg`

| Storage ID | Type | Path / source | Nodes | Content |
|------------|------|---------------|-------|---------|
| `local` | Directory | `/var/lib/vz` | `pve1`, `pve3` | `backup`, `iso`, `vztmpl` |
| `local-lvm` | LVM-thin | local LVM | `pve1`, `pve3` | `images`, `rootdir` |
| `pve1-storage` | ZFS pool | local ZFS on `pve1` | `pve1` | Application data on the infra node |
| `pve2-storage` | (defined) | — | `pve2` | Not part of the active design |
| `nas` | NFS | `192.168.1.16:/data/pve` | `pve1`, `pve3` | `rootdir`, `iso`, `backup`, `images` (`prune-backups keep-all=1`) |

`pve3` has **no native ZFS pool on the host**. Its persistent data lives on `local-lvm`, on TrueNAS datasets, or on the shared `nas`.

## TrueNAS VM

- VM `127` `truenas` runs on `pve3` with multiple physical disks passed through.
- It exposes datasets (NFS/SMB) to the rest of the cluster.
- Dataset-level layout, snapshot schedule, and replication targets are **not captured here**; this page will be extended once the TrueNAS configuration is exported.

## Tier model

This is how the storage layer is used in practice:

| Tier | Where it lives | What sits there |
|------|----------------|-----------------|
| Node-local system | `local`, `local-lvm` on each node | VM and CT root disks, ISOs, templates, local backups |
| Application data (infra) | `pve1-storage` (ZFS on `pve1`) | Persistent data for infra-side workloads |
| Application data (compute) | `local-lvm` on `pve3`, TrueNAS datasets | Persistent data for production workloads |
| Bulk / shared | TrueNAS datasets, `nas` | Media library, large archives, cross-node assets |
| Backup | `nas` (configured), local Proxmox backups in `local` | See [Backups](/architecture/backups) |
