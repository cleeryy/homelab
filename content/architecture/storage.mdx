## Purpose

Describe storage as a platform concern: which tiers exist, what each tier is for, and where service data lives by design.

## Scope

Logical layout and policy. Disk-level details (models, sizes, slot positions) go in [Inventory > Storage assets](/inventory/storage-assets).

## Target state

- Distinct storage tiers by purpose:
  - **System / VM disks** — fast, local to the node.
  - **Application data** — durable, snapshot-capable.
  - **Bulk / media** — high-capacity, throughput-oriented.
  - **Backups** — separate failure domain from the primary data.
- Snapshots are a first-class feature where supported. They are not a replacement for backups.
- Restore is exercised, not just configured.
