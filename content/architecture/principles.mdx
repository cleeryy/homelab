## Documentation principles

- **Current state, not aspiration.** Pages describe what is actually running. In-flight transitions are noted explicitly on the [migration plan](/architecture/migration-plan).
- **Don't invent.** When a value (VLAN ID, backup schedule, port, secret) is not in the source material, it is marked **unknown** or **pending validation**.
- **Use real names.** VM, LXC, and container names are reproduced as they appear in Proxmox or Beszel. Display mismatches (e.g. Beszel hostname vs. system label) are called out, not silently reconciled.
- **One source of truth.** Fumadocs is the canonical documentation surface. Docmost remains visible in the inventory because it is still deployed.

## Operational framing

- **Asymmetric nodes.** `pve1` is the infrastructure and management node; `pve3` is the primary compute and production node. The cluster is not symmetric and HA failover between nodes is not configured.
- **Three execution layers.** Proxmox host → VM/LXC → Docker stack. Documentation is structured accordingly.
- **Shared NFS is shared.** The `nas` mount is reachable from both `pve1` and `pve3` and is configured for `rootdir`, `iso`, `backup`, and `images`.
- **`pve2` is excluded.** It is not part of the active design. The `pve2-storage` entry exists for historical reasons.

## Non-goals

- High availability across nodes.
- Symmetry between `pve1` and `pve3`.
- A single ingress path before the role of `ubuntu-nginx-proxy-manager` vs. `ubuntu-nginxrpm` is confirmed.
