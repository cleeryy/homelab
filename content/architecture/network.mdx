## Edge — OPNsense

OPNsense runs as VM `401` on `pve3`:

- `net0` on `vmbr0` — management LAN side.
- `net1` on `vmbr1` — internal/routed side.

OPNsense is the firewall/router and the boundary between WAN, management LAN, and the internal/routed segment.

## Ingress

Two reverse-proxy VMs are present on `pve1`:

- `ubuntu-nginx-proxy-manager` (`100`) — Nginx Proxy Manager.
- `ubuntu-nginxrpm` (`105`) — second reverse-proxy VM.

Both are visible on the management LAN. The canonical ingress split (which traffic goes through which) is **not confirmed**; this is documented in the [migration plan](/architecture/migration-plan) for resolution.

A `cloudflared` LXC (`103`) on `pve1` provides outbound tunneling for selected services. The scope of services fronted by this tunnel is not captured here.

## Beszel hostname mismatches

Two Docker hosts report a system hostname `ntfy` in Beszel while their system label is different:

- `n8n` host — label `n8n`, system hostname `ntfy`.
- `web-apps` host — label `web-apps`, system hostname `ntfy`.

The cause is not documented. The mismatch is recorded so that operators reading Beszel do not misidentify hosts.
