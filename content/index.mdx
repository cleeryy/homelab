This site is the internal architecture reference for the homelab. It documents the cluster, the workloads currently running on it, the storage and network layout, and the operational notes that go with them.

## Platform at a glance

- Proxmox VE cluster `home`, 9.x line, two active nodes, quorate.
  - `pve1` at `192.168.1.101/24` — infrastructure and management node.
  - `pve3` at `192.168.1.103/24` — primary compute and production node.
- `pve2` is defined in cluster artefacts (including a `pve2-storage` entry) but is not part of the active design.
- Shared storage: NFS share `nas` from `192.168.1.16:/data/pve`, mounted on both active nodes.
- Edge router: OPNsense VM (`401`) on `pve3`, dual-homed on `vmbr0` and `vmbr1`.
- Ingress: Nginx Proxy Manager VM (`100`) and an additional reverse-proxy VM `ubuntu-nginxrpm` (`105`) on `pve1`; a `cloudflared` LXC (`103`) on `pve1` for tunneling.
- Observability: Beszel VM (`202`) on `pve1` collects host and Docker metrics.
- Documentation: this Fumadocs site. Docmost (LXC `109`) is still present from the previous setup.

## Conventions

- Pages describe what is actually running. Where a fact is not in the source material it is marked **unknown** or **pending validation**, never invented.
- IDs and names match Proxmox exactly: VM and LXC IDs are integers, names are reproduced as displayed.
- Docker workloads are documented as services running **inside their VMs**, not at the Proxmox level.
