This site is the source of truth for the homelab. It describes the **target architecture** — what the platform should look like — and the operational practices that keep it healthy. The current state is documented only where it clarifies the migration direction.

## Platform at a glance

- Proxmox VE 9.2 cluster `home`, two active nodes, quorum enabled.
  - `pve3` (`192.168.1.103`) — primary compute and production workloads.
  - `pve1` (`192.168.1.101`) — infrastructure and management plane.
- `pve2` is retained as a disabled historical reference and is not part of the active design.
- Edge routing is handled by an OPNsense VM on `pve3`. User-facing HTTPS ingress is fronted by Nginx Proxy Manager on `pve1`.
- Bulk and shared storage is centralised on TrueNAS (VM on `pve3`), with an NFS share (`nas`) reachable from both nodes.

## Conventions

- Pages describe the **target state** first. Differences from the current state belong under a Notes or migration section, not in the main body.
- Unknown or undecided values are marked **pending validation**, never invented.
- Naming, casing, and structure stay consistent across sections; mirror existing pages when adding new ones.
