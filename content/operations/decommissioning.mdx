---
title: Decommissioning
description: Procedure for retiring a service, guest, or node.
---

## Service inside a Docker host

- [ ] Stop the container in compose.
- [ ] Remove the service block from the compose file.
- [ ] Remove the volume if data is not preserved.
- [ ] Commit the compose change to Forgejo.
- [ ] Update the matching service page under [Services](/services/core/reverse-proxy).

## Standalone VM or LXC

- [ ] Confirm no other service depends on it ([Service placement](/architecture/service-placement)).
- [ ] Remove ingress and DNS entries.
- [ ] Take a final `vzdump` if data must be preserved.
- [ ] Stop the guest.
- [ ] Remove the guest from Proxmox.
- [ ] Remove the host from Beszel monitoring.
- [ ] Update [Inventory](/inventory/virtual-machines) and [Service placement](/architecture/service-placement).

## Node (`pve2`, future)

- [ ] Confirm no workloads, snapshots, or storage references point at the node.
- [ ] Remove the node from the cluster using the documented Proxmox procedure (out of scope here).
- [ ] Remove the matching `<node>-storage` entry from `/etc/pve/storage.cfg`.
- [ ] Update [Nodes](/inventory/nodes).
